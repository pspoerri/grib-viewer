# 04 — Frontend (visualization engine)

React 19 + TypeScript + Vite + MapLibre GL v5. Full port of the
weather-api engine, adapted to the bbox window API. Runtime deps:
`maplibre-gl`, `pmtiles`, `react`, `react-dom` — nothing else. Charts
are hand-rolled SVG, protobuf is a hand-rolled DataView reader, state
is React + URL hash.

## State model

- **The layer list (`MapLayer[]`) is the single source of truth.**
  Every choice — variable, display mode, ensemble product (id
  suffix), window op, colormap, opacity, threshold, lapse flag —
  lives on a layer, so the whole map round-trips through the URL hash
  (`encode/decodeMapHash`): model, run, preset, layers, basemap,
  projection, camera, windowMode, time anchor, point of interest.
- `usePersistentState` (localStorage) for cosmetic prefs: unit prefs,
  time format (Local/UTC/Lead), playback speed, HDR.
- No state library. `App.tsx` owns state; an imperative
  `WeatherMapHandle` (`setPlayhead`, `setPlaying`, `waitForFrameReady`,
  `setView`, `flyTo`) lets the TimeBar drive the vsync playback loop
  outside React.

## Data plane

- Hand-rolled protobuf decode of the `Window` message; animation
  chunks expose per-frame `Int16Array` views sharing one buffer.
- **One `/data` request per (layer, viewport, frame)** with the
  viewport bbox (+ one polar-band request per visible pole on the
  globe). No tile stitching. `maxcells` = 700k·dpr², doubled by the
  HDR toggle.
- LRU window cache keyed by data identity
  (`model|run|var|bbox|frame|agg|windowSpan`), sized to the playback
  working set; AbortControllers cancel superseded fetches on
  pan/zoom/layer change. Chunked prefetch (8 h ahead) while playing.
- Debounced re-fetch on moveend (the bbox changes with every pan —
  the price of bbox windows; the cache + debounce keep it smooth).

## GPU render core (`WxV2Layer`, a MapLibre CustomLayerInterface, WebGL2)

- Field uploaded as **R32F texture** (int16 dequantized, NaN =
  nodata). Subdivided mesh quad over the window's mercator bbox; the
  **fragment shader inverse-projects** pixel → lat/lon → fractional
  grid texel (mercator is nonlinear; a flat UV quad stretches).
- Extended-mercator poles (linear-in-latitude caps beyond ±85.05°) so
  the globe covers poles; shared `MERC_DECODE` GLSL.
- Globe-aware vertex shader: sphere lift, back-hemisphere clip cull,
  mercator↔globe transition lerp.
- Sampling `u_interp`: nearest / NaN-aware bilinear / bicubic
  B-spline, weights renormalized over valid taps.
- **Colormap in-shader** from a 256-entry 1D texture; linear t, log
  for precip palettes (floor vmin>0 else vmax/1000). Temperature
  stepping is baked into the ramp texture (bands: 2 °C below −30,
  1 °C in [−30,40], 2 °C in [40,50], 5 °C above), aligned with the
  legend.
- **GPU frame tween**: two frame textures + `u_mix`, NaN-aware — the
  smooth-playback mechanism.
- Every shader formula (colormap stepping, log t, lapse, terrarium
  decode, B-spline kernel) has a unit-tested JS mirror used by
  legend/labels — drape and legend can never disagree.

### Layer display modes

`tiles | contour | value | barbs | flow`

- **tiles** — the drape above.
- **contour** — GPU shader contours: second program sharing the
  drape's textures; 16-tap cubic B-spline reconstruction (value +
  analytic gradient, low-passes regridder noise), `fwidth`-AA
  signed-distance isolines, optional fill. Interval auto from p2–p98
  range (~8 nice steps) or explicit (geopotential 100 gpm, labels in
  dam). No labels (non-goal).
- **flow** — GPU-resident wind streamlines: particle state in an
  RGBA32F ring-buffer texture (trail 64), ping-pong advection per
  vsync lerping two (u,v) field slots, AA line-strip trails.
  `flowParticles` (2000 default / 8000 presets), speed calibration
  10 CSS px/s at 10 km/h. On composites, fetch from a single covering
  contributor (particles can't cross seams).
- **value** — `/grid` GeoJSON → symbol layer with unit-converted text
  labels, screen-px spacing.
- **barbs** — u+v `/grid` pair merged client-side, canvas-rasterized
  barb glyph per speed bucket, `icon-rotate` by direction.

### Composite (client-side blending)

`/api/composite/{auto|auto_eps}` ladder → one draw unit per
contributor, finest on top, `is_base` at the bottom:

- **Edge feather**: alpha ramp to 0 over 50 km inside each finer
  contributor's footprint (equirectangular distance-to-edge).
- **Per-pixel finest-wins**: coarser drapes yield inside finer *active*
  domains (exclusion textures, ≤6), gated per-pixel on the finer
  drape's own value texture being valid, ramped over feather-depth
  F..2F for constant coverage.
- A contributor past its horizon / not covering the active window
  drops out so the coarser one shows through.
- Per-layer det/eps routing (`layerModel`); the animation timeline is
  globally owned by the selected model.

## Time control

- TimeBar: per-day scrubbing (tz-aware grouping), hour ticks from the
  smallest step gap; window modes render one segment per block.
- `windowMode ∈ {hourly, 3h, 6h, 12h, daily}` — calendar-bucketed
  (6h@00/06/12/18 etc.), DST-safe, tz-aware; ops ride the variable id
  (`__{N}h_{op}`); per-layer op from catalog default. Precip "Total"
  selects `precip_{N}h` variables, not an op.
- Playback: vsync rAF loop, fractional `setPlayhead` → GPU tween
  (hourly) or block stepping (windowed); default 250 ms per forecast
  hour; gates on `isFrameReady` (never tweens into half-loaded data);
  1 s dwell on the last frame.
- **Start anchor**: default the playhead to round-down(now), clamped
  past de-accumulation null frames; frames before it are recent past
  weather. Disabled on `synthetic_time` runs (anchor = first step,
  Lead display forced).
- **Run browser** (new): a panel listing `/runs` per model with
  validity-window coverage bars per variable; picking a run pins
  `?run=` on all requests and shows a "pinned run" chip. This is the
  timerange-exploration surface.

## Ensemble UX

- **DET | EPS master switch** (composite header) bulk-applies per-layer
  modes; picking an EPS product on det auto-switches (never back).
- **Per-layer product picker**: `Med · Mean · p10 · p90` + overflow
  (Control, p25, p75, Min, Max, Spread, Chance-of, Member N); the
  product is a suffix on `layer.variable`; segments gate on the
  model's advertised products.
- **Chance-of legend**: draggable threshold marker on the colorbar,
  dimmed non-event side, ≥/≤ chip, in-mode unit cycler, hover value
  bubble, readout ("map shows: chance that T₂ₘ ≥ 24 °C"); curated
  default thresholds (t_2m 20 °C, gusts Bft 7, wind Bft 5, precip
  1 mm/h, ghi 400 W/m²) → remembered → mid-domain; drag debounced
  ~200 ms into a `_gt{V}{unit}` id rewrite. Dedicated `prob`
  colormap, transparent at 0.
- Spread renders through the normal colorbar with numeric min/max.

## Legends & units

One legend row per visible layer: colormap bar (drawn client-side
from `/colormaps` stops), min/max labels, window-mode selector,
window-op chips, product picker, threshold UI, lapse ⛰ toggle. Unit
groups (°C/°F/K, km/h·m/s·kn, hPa, mm/in, m/ft, gpm/dam) cycle per
group, persisted; labels convert, tile colors don't.

## Point interactions

- Click → popup: reverse geocode (**Nominatim** `/reverse`,
  `format=jsonv2`, zoom-scaled acceptance), per-variable point series
  via `/point` spans.
- **Meteogram** (hand-rolled SVG): temperature line + p10–p90 band,
  three-band cloud cover, precip bars + >1 mm/h probability overlay,
  pmsl panel, active-product panel, day/night shading from solar
  elevation (suppressed on synthetic time), wind-barb percentile
  strip. Daily summary strip computed client-side from the series
  (type-aware reducers: temp min/max, gust max, flux mean,
  accumulation sum).
- Multi-model compare tab: same point across all physical models,
  fixed color per model, EPS bands where advertised.
- **Location search**: Nominatim `/search` (`format=jsonv2`,
  `addressdetails=0`, debounced, `email`/UA per usage policy), ranked
  candidates, fly-to with bbox-aware zoom.

## Presets & layer builder

- Topics on a PresetBar: **Temperature, Surface, Upper air, Custom**;
  upper-air has a shared 850/500/300 hPa level selector that rewrites
  every level-bearing layer id and persists across phenomenon
  switches.
- Presets = named `MapLayer[]` (wind+flow, t_2m variants, precip
  stack with hidden alternates, pressure contour+fill, snow, CAPE,
  radiation, humidity, upper-air height/temp/jet).
- User presets persist to localStorage; active-preset detection
  ignores isobaric level differences.
- Layer builder (hamburger): visibility, mode, opacity, product,
  colormap, agg op, add/remove; model+run selectors, basemap,
  projection (globe default), Local/UTC/Lead, HDR, playback speed.

## Basemap & terrain

- **Basemap**: pmtiles-backed vector styles from `tiles.rsp.li`,
  **copying its style rules** — the black/dark/grayscale/light/white/
  summer/winter family, each split into *back* (land, landuse,
  hillshade) and *front* (borders, water lines, labels) documents
  merged client-side; weather drapes insert at the recorded
  front-anchor so labels stay on top. Style JSONs are vendored into
  the repo (fetched once from tiles.rsp.li) so the app also runs
  against a self-hosted pmtiles file.
- **Terrain**: Mapterhorn, per their terrain example
  (https://mapterhorn.com/examples/terrain/) — a `raster-dem` source
  from the Mapterhorn PMTiles (terrarium encoding,
  `meters = R·256 + G + B/256 − 32768`), used three ways:
  1. MapLibre `setTerrain` 3D exaggeration + hillshade layer (the
     example's pattern);
  2. globe terrain drape — weather mesh lifted onto the DEM (surface
     fields glued, upper-air floated) with Lambert hillshade;
  3. **client-side lapse correction**: screen temps corrected in-shader
     `T_site = T_model + γ·(z_site − z_model)`, γ = −6.5 K/km; z_site
     from the Mapterhorn DEM mosaic, z_model from the window's height
     plane, sampled with the same kernel to avoid cell-boundary halos.
     Per-layer ⛰ toggle. DEM failure latches the feature off after 3
     misses.
- Optional satellite-imagery underlay toggle (ESA WorldCover style
  from tiles.rsp.li).

## Testing

No framework: `node --test`-style plain assert scripts over every pure
module (time bucketing, id grammar helpers, products, window math,
protobuf decode, colormap/log/stepping mirrors, B-spline kernel
partition-of-unity, lapse math, hash round-trip). GPU/React verified
by headless-browser QA only.

# wetter — standalone NWP GRIB viewer

Clean-room reimplementation of the ideas in the retired `weather-api`
project, re-shaped around one goal: **point the stack at GRIB output —
an S3 bucket, a web host, or a plain directory — and explore it on a
map**, with the full visualization engine (GPU field drapes, shader
contours, wind streamlines, ensemble products, meteograms) but **no
ingest-to-archive pipeline**. All fields are derived online from
locally buffered GRIB files.

## Product goals

1. Standalone experience to look at GRIB data from NWP models.
2. Multiple data sources and locations at once (operational feeds and
   ad-hoc run folders side by side).
3. Live operational output and debug/experimental output mix freely —
   including runs whose timestamps are synthetic or whose filenames
   follow no convention (index by GRIB headers, not names).
4. Explore available time ranges: multiple retained runs, a run
   browser, per-variable horizon coverage.
5. Derive **all** fields online: ensemble percentiles, exceedance
   probabilities, de-accumulation, wind speed, all derived diagnostics
   — computed at request time from decoded GRIB, cached in memory.
6. Full visualization engine port (see spec 04).

## Non-goals (deliberate cuts vs weather-api)

- No `.wxt`/`.wxt2` archive formats, no `wx-ingest` binary, no
  pre-built percentile/dist/prob archives. The GRIB buffer is the only
  on-disk artifact.
- No PNG map tiles, no z/x/y datatile endpoint — **bbox windows only**
  (one request per viewport, efficient against a GRIB-backed server).
- No satellite stack (GOES/Himawari), no MCP server, no `/summary`
  daily endpoint in v1 (daily reductions happen client-side from point
  series; revisit if a consumer needs it server-side).
- No backend elevation DEM / `wx-elevation`. Lapse-rate correction is
  client-side only (GPU, Mapterhorn terrarium tiles); backend `/point`
  returns raw model values plus model surface height.
- No geocoder ownership — the frontend calls an external geocoder URL.

## Architecture

```
MeteoSwiss STAC ─┐
DWD opendata ────┤                       ┌─▶ /api/... (JSON, protobuf windows)
S3 bucket ───────┼─▶ source adapters ─▶ GRIB buffer ─▶ derive engine (in-memory caches)
web host ────────┤    (discover runs,    (data_dir,        │
local folder ────┘     fetch loop/once)   decompressed)    ▼
                                                     React + MapLibre GL frontend
                                                     (GPU drape, contours, flow,
                                                      Mapterhorn terrain, rsp.li styles)
```

One Go binary, `wetter`, with subcommands:

- `wetter serve --config wetter.yaml [--fetch]` — HTTP API; with
  `--fetch` also the configured fetch loops in-process.
- `wetter fetch --config wetter.yaml [--once] [--source id]` — run
  discovery/download standalone (for teams that only want the buffer).

Frontend: TypeScript, React, MapLibre GL, Vite. No chart library, no
protobuf runtime, no state library — hand-rolled SVG charts, hand-rolled
protobuf reader, React state + URL hash (proven in the old system, keeps
the dependency surface tiny).

Backend: Go, stdlib-first. Dependencies: `pspoerri/go-tiled-eccodes`
(pure-Go GRIB2 decode), `klauspost/compress/zstd` (only if response
compression beyond gzip is wanted — start without), `gopkg.in/yaml.v3`
(config), `prometheus/client_golang` (metrics). No CGO anywhere.

## Cross-cutting invariants (carried over from weather-api, hard-won)

- **Reduce before regrid.** Ensemble statistics are computed per cell
  on the native grid across members, then regridded. Never the reverse.
- **Percentile-of-speed ≠ speed-of-percentiles.** Wind speed is
  `hypot(u,v)` per member (paired by perturbation number), then
  reduced. A speed product derived from component percentiles is wrong
  and must be refused.
- **At-least-half rule.** A cell needs ≥ ⌈(n+1)/2⌉ valid members to
  produce an ensemble statistic; otherwise NoData.
- **Time is addressed by absolute instants** (RFC3339) end to end, so
  contributors with different cadences reconcile; runs without a real
  wall-clock anchor still get an internal absolute axis (see 01).
- **The layer list is the frontend's single source of truth**; every
  product/mode choice encodes as a suffix or field on a layer, so the
  whole map state round-trips through the URL hash.
- **Drape and legend never disagree**: every shader formula (colormap
  stepping, log scaling, lapse) has a JS mirror that is unit-tested and
  used for the legend/labels.
- **Atomic writes**: any published file goes through temp + rename.
- **int16 quantization** with per-variable scale/offset, −32768 =
  NoData, for wire payloads (spec 05).

## Phasing

1. **Core**: folder source + one DWD model, deterministic only; window
   endpoint; frontend drape + timebar + point popup.
2. **Ensemble**: member decode, reductions, product grammar, ensemble
   UX, meteogram bands.
3. **Sources**: MeteoSwiss STAC, S3/web-index sources, fetch daemon,
   run browser.
4. **Full engine**: contours, flow layer, composite ladder, presets,
   lapse/terrain, barbs/value labels.

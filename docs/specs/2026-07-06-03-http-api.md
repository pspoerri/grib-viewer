# 03 ‚Äî HTTP API

All endpoints under `/api`. Handlers are thin: parse ‚Üí derive-engine
`Window`/`Sample` primitive ‚Üí encode. Responses gzip'd; CORS
configurable origin list (default loopback only).

## Endpoint surface

| Method + Path | Purpose |
|---|---|
| `GET /api/healthz` | Liveness |
| `GET /api/status` | Per-source fetch status (run, progress, last error) |
| `GET /api/models` | Catalog: sources/models + servable variables + products |
| `GET /api/models/{model}/runs` | **All buffered runs** with valid windows + per-var step coverage (the timerange explorer) |
| `GET /api/models/{model}/runs/{run\|latest}` | One run: valid_from/to, horizon, cadence, complete, synthetic_time |
| `GET /api/models/{model}/meta/{var}?run=` | Variable metadata: grid def, pyramid, steps present, units, quantization, colormap, products |
| `GET /api/models/{model}/point/{time}/{var}?lat=&lon=&run=` | Bilinear point value; series when `{time}` is a span |
| `GET /api/models/{model}/data/{time}/{var}?bbox=&maxcells=&run=` | **The window endpoint** ‚Äî protobuf native-grid window; span ‚Üí multi-frame animation chunk |
| `GET /api/models/{model}/window/{time}/{var}?bbox=&run=` | Same as `/data` but JSON (debug/inspection) |
| `GET /api/models/{model}/grid/{time}/{var}?bbox=&spacing=&run=` | GeoJSON point lattice (value labels, barbs) |
| `GET /api/composite/{auto\|auto_eps}` | Contributor ladder for client-side compositing |
| `GET /api/colormaps` | Colormap catalog (name, stops, hidden) |
| `GET /api/openapi.json` | OpenAPI 3 spec (coverage-tested against the mux) |
| `GET /metrics` | Prometheus |

**No z/x/y tile endpoint.** The frontend issues **one `/data` request
per (layer, viewport, frame)** with the viewport bbox ‚Äî dramatically
fewer queries than tiling, and each request maps to one contiguous
window read over the derived native grid. Pole-crossing globe views
issue one extra polar-band bbox request per pole. `maxcells` (default
700 000) is the client's cell budget; the server picks the pyramid
level, so retina/HDR clients simply send a bigger budget.

No `style.json` endpoint: the frontend owns its MapLibre style
(basemap from tiles.rsp.li, spec 04); data layers are custom WebGL
layers fed by `/data`, so there is nothing for the server to style.

## Multi-run addressing

Every data endpoint takes `?run=` (RFC3339 or the run id from
`/runs`); default `latest`. This replaces latest-only retention and
makes the run browser work.

## `{time}` grammar

- `latest` ‚Äî first step of the run (analysis).
- `now` ‚Äî wall-clock rounded to the hour (rejected with 400 on
  `synthetic_time` runs).
- RFC3339 instant ‚Äî nearest native step; more than one cadence step
  outside the axis ‚Üí 404 naming the valid `[from, to]`.
- Span `{start}+{ISO8601-duration}` (`2026-07-06T12:00:00Z+PT6H`;
  duration subset: weeks or D/H/M/S, no months/years). Split on the
  **last** `+` whose tail starts with `P` (tz offsets contain `+`).
- Lead-time form `+{N}h` ‚Äî resolves against the run's reference time;
  the natural form for synthetic-time runs (`/data/+12h/t_2m`).

Semantics with window-op ids (`__{N}h_{op}`, below): instant ‚Üí one
trailing inclusive block `[t‚àíN, t]`; span ‚Üí `‚åäM/N‚åã` half-open forward
blocks (animation). Plain id + span ‚Üí every native step (chunk).
Span + plain id on single-frame endpoints (grid) ‚Üí 400.

## Variable-id grammar

Composition: `{base}{level}{plane}{__window}[unit]` ‚Äî parsed by
stripping in reverse order:

1. **`[unit]` bracket** ‚Äî display conversion, applied last
   (`t_2m[F]`). Families: temperature K/C/F, speed ms/kmh/mph/kn/bft,
   precip mm/in, pressure Pa/hPa, length m/ft, geopotential gpm/dam.
2. **`__{N}h_{op}`** window suffix, `op ‚àà max|min|mean|sum`
   (`t_2m__24h_max`). Double underscore disambiguates from the
   ensemble `_mean`. Exceedance ids take bare `__{N}h` = peak hourly
   probability (documented lower bound); accumulant exceedance over a
   window uses the per-member **window-total** `P(Œîacc ‚â• thr)`.
3. **Plane/product suffix**:
   - `_p{N}` percentile (any 0‚Äì100; `_p0` min, `_p100` max), bare id
     = p50 on ensemble vars, the value itself on deterministic.
   - `_mean`, `_ctrl`, `_spread` (= p90‚àíp10).
   - `_m{N}` individual member (debug-friendly: view any raw member).
   - Exceedance `_gt{V}{unit}` / `_lt{V}{unit}`, decimal comma as `p`
     (`tot_prec_gt2p5mm`, `t_2m_lt0c`, `vmax_10m_gtbft8`). Unit
     tokens: c/f/k, ms/kmh/kt/bft, mm/in, hpa, w. Values are
     fractions 0..1.
4. **Level**: soil `_l{N}` (`t_so_l3`); isobaric `_{lvl}hpa`
   (`t_850hpa`, `fi_500hpa`, `wind_500hpa`) ‚Äî one grammar, the
   `_l{hPa}` form from the old docs is dropped.

A requested product the run cannot support (e.g. `_p90` on a
deterministic run, speed percentiles without paired members) is a hard
404 ‚Äî never a silent fallback.

## Protobuf window (`/data`)

```proto
message Grid { int32 nx; int32 ny; double lat0, lon0, dlat, dlon; }
message Window {
  string model; string variable; Grid grid;
  bytes  values;              // LE int16 row-major, nx*ny*2 per frame
  float  scale; float offset; int32 nodata;   // value = raw*scale+offset
  bytes  height;              // optional z_model int16 plane (lapse)
  int32  nframes;             // >1 = animation chunk
  repeated int64 frame_unix;  // valid time per frame
  bool   synthetic_time;      // frame_unix not wall-clock meaningful
  int64  run_unix;            // reference time (lead-time display)
}
```

- Quantization: per-variable scale/offset (spec 05); NaN ‚Üí nodata
  sentinel (‚àí32768); finite values clamped to ¬±32767.
- The **quantization actually used is declared in the message** (a
  derived field may ride its base's scale) ‚Äî the client must never
  assume catalog scale.
- Chunk cap 48 frames; missing steps are skipped, not padded.
- Margin: +2 native cells beyond bbox for client interpolation.

## Caching

- Concrete `{time}` + explicit `?run=` ‚Üí strong ETag
  `"{run}|{path}?{query}"`, `Cache-Control: public, max-age=3600,
  immutable`, `If-None-Match` answered before any decode.
- Mutable tokens (`latest`, `now`, defaulted run) ‚Üí no store.
- `X-Model-Run` header on every data response.

## Composite ladder (`/api/composite/{id}`)

`auto` (deterministic planes) and `auto_eps` (ensemble): returns the
buffered contributors finest‚Üícoarsest ‚Äî
`{model, native_deg, bbox, is_base, run, horizon_to}` + synthetic run
id (hash of contributor run tuples). **All blending is client-side**
(spec 04); the backend only publishes the ladder, built from the
configured sources' resolutions and footprints. Sources with
`synthetic_time` runs are excluded. Point requests against a composite
id resolve to the finest covering contributor, two-pass for windowed
ids (a rung must cover the whole block, else next rung ‚Äî the
"daily-max shows nightly-min" bug class).

## Errors

Structured JSON `{error, valid_from?, valid_to?}`; grammar problems
400, out-of-axis/unsupported-product 404, span-needs-op 400. Errors
name the fix (valid window, supported products list) ‚Äî a client should
self-correct in one round trip.
